# /note-refine-titles

Generate hierarchical titles for text content based on semantic analysis.

## Usage

```
/note-refine-titles {input_file_path}
```

## Output

优化后的完整markdown内容（UTF-8编码）直接输出到标准输出，不修改原文件。

## 核心特性

### 严格的原文保护
- **零修改原则**：绝对不修改、删除或增加任何原文内容
- **格式保持**：保持原文的所有标点符号、空格、换行等格式细节
- **语义完整**：确保原文的逻辑结构和论证链条完整无缺

### 智能语义换行
- **识别过长的段落**：对明显过长的段落进行合理分段
- **在自然断句处换行**：优先在句号、问号、感叹号等完整句子结束处换行
- **保持语义完整**：确保换行不破坏句子的完整性和连贯性

### 精准的层次分析
- **主题聚类**：使用语义分析将相关内容合理分组，支持一个小节内包含多个相似或关联度较强的主题
- **逻辑关系识别**：识别时间顺序、因果关系、重要性层次等
- **平衡性控制**：确保各级标题下的内容量相对均衡

## Requirements

1. **严格保持原文完整性** - 只添加或替换标题，**绝对不得修改、删除或增加任何原文内容**，包括文字、标点符号、格式等所有细节
2. **智能分块处理** - 对于长文档（超过20000字符），采用智能分块处理机制，确保完整覆盖所有内容
3. **语义换行优化** - 对于明显过长的段落，可在自然断句处合理分段，提高可读性，但不得改变原文意义
4. **基于语义的智能层级选择** - 根据内容的语义复杂度和逻辑结构选择合适的层级深度：
   - **语义边界优先**：在主题转换、逻辑转折处自然划分章节
   - **最小单元完整**：每个标题下至少包含一个完整的论述单元（观点+支撑+结论）
   - **结构合理性**：避免过度细分，确保每个标题代表有意义的内容块
   - **认知友好性**：层级深度控制在3-4层以内，符合人脑理解和记忆规律
   - **字数要求约束**：严格遵守字数下限要求，每章不少于1500字，每节不少于500字
5. 使用指定的编号模式
6. 输出结果到标准输出（UTF-8编码）
7. **柔性约束指导原则**：创建符合语义逻辑的标题结构：
   - **最小内容单元**：确保每个标题下的内容构成完整的认知单元，避免出现只有一个段落的小节
   - **字数约束优先**：严格遵循字数下限，每章不少于1500字，每节不少于500字，内容不足时优先通过段落内部分换行解决可读性问题
   - **段落内部细分优先**：当内容过短无法满足独立成节时，应在现有小节内部通过语义换行进行细分，而不是创建新的小节
   - **同级标题平衡**：同级标题下的内容量和重要性应相对平衡，建议3-8个同级标题
   - **语义相关性**：同级标题下的内容必须具有较强的逻辑相关性
   - **防止过度细分**：避免一章下面出现过多小节，一般建议每章3-6个小节
8. **语义完整性优先原则**：
   - 当内容主题简洁时，不强制拆分，保持自然的内容完整性
   - 避免为追求标题数量而人为分割连贯的论述
   - 确保每个标题都准确反映其下内容的核心主题
   - 优先考虑内容的逻辑流畅性和读者理解体验
   - **字数底线保障**：宁可让节内容更长一些，也要确保满足每章≥1500字、每节≥500字的最低要求

## 智能分块处理机制

### 分块触发条件
- 当输入文档超过 **15000字符** 时，自动启用智能分块处理
- 每个分块大小控制在 **15000-20000字符** 以内
- 确保有足够的处理空间，同时避免过度分割

### 分块算法
1. **语义边界检测**：优先在段落、现有章节等自然边界处分割
2. **上下文保持**：确保每个分块包含足够的上下文信息
3. **编号连续性**：维护分块间标题编号的连续性和一致性
4. **重叠处理**：在分块边界处保留少量重叠内容（50-100字符），确保语义连贯性

### 处理流程
1. **文档分析**：检测文档长度，确定是否需要分块
2. **分块分割**：按照语义边界将文档分割成多个块
3. **逐块处理**：为每个分块生成标题，维护上下文信息
4. **结果合并**：智能合并各块处理结果，重新编号标题
5. **质量验证**：确保处理结果的完整性和一致性

## Title Hierarchy Rules

- **Document title**: `# {总标题内容}` (main title for the entire document)
- **Level 1 sections**: `## 1`, `## 2`, `## 3`, etc.
- **Level 2 subsections**: `### 1.1`, `### 1.2`, `### 2.1`, etc.
- **Level 3 sub-subsections**: `#### 1.1.1`, `#### 1.1.2`, `#### 2.1.1`, etc.

## Implementation Notes

### 文本处理原则
- **严格保护原文内容**：采用只读方式处理文本，确保不修改任何文字、标点、格式
- **智能语义分析**：通过自然语言处理技术识别主题转换和逻辑边界
- **保持原文风格**：保留原作者的语言风格、表达习惯和叙述方式
- **维护内容连贯性**：确保添加标题后文本的逻辑流畅性和可读性

### 语义换行处理
- **识别过长的段落**：对明显过长的段落进行合理分段
- **在自然断句处换行**：优先在句号、问号、感叹号等完整句子结束处换行
- **保持语义完整**：确保换行不破坏句子的完整性和连贯性
- **字数保障原则**：在进行语义换行时，要确保满足每章≥1500字、每节≥500字的字数下限要求

### 语义驱动的层次结构生成
- **字数约束优先原则**：严格遵守字数下限要求（每章≥1500字，每节≥500字），在满足字数要求的前提下进行语义划分
- **多主题小节支持**：鼓励在一个小节内组织多个相似或关联度较强的主题，通过主题聚合满足字数要求
- **主题聚类分析**：使用语义相似度算法将相关内容合理分组，优先合并小内容而非拆分大内容
- **主题关联度判断**：
  - **语义相似性**：使用相同的术语或概念的主题可以合并
  - **逻辑相关性**：具有因果关系、对比关系、补充关系等强逻辑关联的主题可以合并
  - **上下文连贯性**：内容自然衔接，不需要过多过渡的主题可以合并
- **多主题小节组织方式**：
  - **并列型**：多个平行的相似主题并列呈现
  - **递进型**：从基础到进阶的相关主题递进排列
  - **对比型**：相互对比或补充的主题组织在一起
  - **总分型**：一个主主题统领多个子主题
- **语义边界识别**：在重大主题转换、逻辑转折、论述层次跳跃的自然边界处划分章节，但优先考虑主题聚合和字数约束
- **逻辑关系识别**：识别因果关系、时间顺序、对比关系、重要性层次等
- **最小单元控制**：确保每个标题下至少包含一个完整的论述单元，避免过度细分，同时满足字数下限
- **结构合理性优化**：
  - 同级标题数量建议控制在3-8个，避免过多或过少
  - 层级深度建议控制在3-4层，符合认知习惯
  - 防止出现"一章一小节"或"一章十几个小节"的不合理结构
  - **字数平衡控制**：确保每章不少于1500字，每节不少于500字，内容不足时优先考虑合并而非拆分
  - **主题聚合优先**：优先通过主题聚合而非章节拆分来满足表达需求
- **语义完整性保证**：当内容主题简洁时，不强制拆分，保持自然的内容完整性
- **编号一致性**：严格按照1, 1.1, 1.1.1的格式进行编号
- **段落内部细分机制**：当发现内容可以进一步细分但会导致字数不足时，在现有小节内部通过语义换行实现可读性优化

### 质量保障措施
- **内容完整性检查**：生成后对比原文，确保无内容丢失或修改
- **格式正确性验证**：检查markdown格式和编号规则的正确性
- **语义一致性确认**：确保标题准确反映对应内容的主题
- **字数约束检查**：
  - **章节字数验证**：确保每个章（##级别）不少于1500字
  - **小节字数验证**：确保每个小节（###级别）不少于500字
  - **字数调整策略**：对于字数不足的情况，优先考虑合并相邻小节，其次在节内进行语义换行优化
- **可读性优化**：通过合适的分段和换行提升整体可读性
- **语义边界与结构质量检查**：
  - **语义边界验证**：确认每个标题都位于自然的语义边界处，重大主题转换合理，同时满足字数要求
  - **最小单元验证**：确保每个标题下包含完整的论述单元，避免只有一个段落的小节，同时达到字数下限
  - **多主题小节验证**：
    - 验证同一小节内的多个主题是否具有强关联性
    - 检查主题组织方式是否合理（并列型、递进型、对比型、总分型）
    - 确认主题之间过渡自然，逻辑连贯
  - **结构合理性评估**：
    - 检查同级标题数量是否合理（建议3-8个）
    - 评估层级深度是否适当（建议不超过4层）
    - 验证每章小节数量是否均衡（建议每章3-6个小节）
    - **字数分布合理性**：检查章节之间的字数分布是否相对均衡
  - **主题聚合效果检查**：
    - 验证主题聚类是否合理，是否存在可以进一步聚合的细分主题
    - 检查是否充分利用了多主题小节的优势来满足字数要求
    - 确认主题关联度判断标准的应用效果
  - **过度细分检查**：识别并修正不合理的内容拆分，重点检查是否存在可以通过主题聚合优化的细分小节
  - **语义连贯性验证**：确保标题结构符合内容的自然逻辑流，多主题小节内部连贯性良好
  - **段落内部细分效果检查**：验证在节内通过语义换行实现可读性优化的效果

### 特殊情况处理
- **长文本无标点**：对于缺少标点的长文本，根据语义主题进行合理分段
- **混合内容类型**：处理包含叙述、说明、论证等多种表达方式的文本
- **专业术语密集**：在技术文档中保持专业术语的准确性和一致性
- **口语化表达**：在口语化文本中保持自然流畅的表达方式

### 输出格式要求
- **标准输出**：将处理后的完整 markdown 内容以UTF-8编码直接输出到标准输出
- **标题层次清晰**：确保各级标题的层级关系明确
- **内容分布均匀**：避免某些部分过长而其他部分过短
- **逻辑结构完整**：保持原文的逻辑结构和论证链条
- **阅读体验优化**：通过合理的分段和换行提升阅读体验

## 具体实现步骤

### 第一步：文档分析
```markdown
请分析以下文档的长度和结构：
{{input_text}}

如果文档不超过15000字符，请直接进行标题优化处理。
```

### 第二步：分块处理（仅在需要时执行）
```markdown
根据文档分析结果，请将文档按照以下规则分割成多个块：

1. 每个分块控制在15000-20000字符以内
2. 优先在段落边界、现有章节标题等自然边界处分割
3. 在分块边界保留50-100字符的重叠内容
4. 为每个分块添加分隔标识：=== 分块 X ===

分割后的分块格式：
=== 分块 1 ===
[分块内容]
=== 分块 1 结束 ===

=== 分块 2 ===
[分块内容]
=== 分块 2 结束 ===
```

### 第三步：逐块标题优化
```markdown
对每个分块进行标题优化处理：

处理分块 {{chunk_number}}/{{total_chunks}}：

=== 分块 {{chunk_number}} 开始 ===
{{chunk_content}}
=== 分块 {{chunk_number}} 结束 ===

要求：
1. 保持原文完整性，只添加或替换标题
2. 根据上下文调整标题编号，确保连续性
3. 继承前面分块的标题结构和编号
4. **字数约束优先**：严格遵守字数下限要求，每章不少于1500字，每节不少于500字
5. **语义完整性控制**：确保每个标题下包含完整的论述单元，避免过度细分
6. **结构合理性指导**：
   - 每个标题下至少包含一个完整的论述单元（观点+支撑+结论）
   - 严格避免出现只有一个段落或字数不足的小节
   - 同级标题数量建议控制在3-8个
   - 层级深度建议不超过4层
7. **多主题聚合策略**：优先将相似或关联度强的主题合并到同一个小节中，通过主题聚类满足字数要求
8. **段落内部细分优先**：当内容可以进一步细分但会导致字数不足时，在现有小节内部通过语义换行实现可读性优化
9. **字数保障策略**：内容不足时优先考虑合并相邻小节，而非创建新的更小的小节
10. **主题关联度应用**：在划分章节时，优先考虑主题间的语义相似性、逻辑相关性和上下文连贯性
11. **语义边界与字数平衡**：在重大主题转换和逻辑转折处划分章节时要考虑字数约束，确保满足最低字数要求
12. **认知友好性**：确保结构符合人脑理解和记忆的规律，多主题小节组织方式清晰易懂

请输出该分块的标题优化结果。
```

### 第四步：结果合并与最终优化
```markdown
将所有分块的处理结果合并，并进行最终优化：

合并规则：
1. 移除分块分隔符
2. 确保标题编号的连续性和一致性

6. **多主题小节验证**：
   - 检查每个小节是否充分利用了主题聚类优势

   - 确认主题关联度判断标准的应用效果

   - 验证每个标题都位于自然的语义边界处
   - 确认每个标题下包含完整的论述单元
   - 检查同级标题的数量和结构是否合理
8. **结构合理性优化**：
   - 识别并修正过度细分或过度合并的情况
   - 确保层级深度适当（建议不超过4层）
   - 验证每章小节数量均衡（建议每章3-6个小节）
   - **字数分布优化**：调整章节划分确保字数要求的满足
   - **主题聚合优化**：进一步优化主题聚类，充分利用多主题小节优势
9. **最终字数保障**：
   - 对于仍然字数不足的小节，考虑与相邻小节合并或进一步主题聚合
   - 对于过长的小节，检查是否可以在内部通过语义换行优化可读性
   - 确保最终结构既满足字数要求又保持语义合理性

请输出最终的完整markdown文档。
```

## 执行指令模板

当用户使用 `/note-refine-titles {input_file_path}` 时，按以下步骤执行：

1. **读取并分析文档**：检测文档长度，确定处理模式
2. **短文档处理**（≤15000字符）：
   - 直接应用标题优化规则，严格确保每个标题层级的内容量要求
   - 强制执行字数约束：每章≥1500字，每节≥500字
   - 优先通过合并而非拆分来满足字数要求
3. **长文档处理**（>15000字符）：
   - 启用智能分块机制
   - 按语义边界分割文档，同时考虑字数约束
   - 逐块处理并维护上下文，严格控制每个标题层级的最小内容量
   - 合并结果并优化结构，强制修正内容不足的标题单元
   - 在分块处理时确保每块大小能够支持满足字数要求的章节划分
4. **最终字数验证与调整**：
   - 检查所有章节是否满足字数要求（每章≥1500字，每节≥500字）
   - 对不满足要求的部分进行合并或优化
   - 确保最终结果既满足字数约束又保持语义合理性
5. **输出最终结果**：确保完整性、一致性和内容合理性，以及字数要求的满足

### 长文档处理示例（>15000字符）

#### 处理流程演示

**第一步：文档分析**
```
文档长度为16500字符，将启用智能分块处理。
```

**第二步：分块分割**
```
=== 分块 1 ===
[前16500字符的内容，包含完整的语义单元]
=== 分块 1 结束 ===

=== 分块 2 ===
[后18000字符的内容，与前一个分块有50-100字符重叠]
=== 分块 2 结束 ===
```

**第三步：逐块处理**
```markdown
# 文档总标题

## 1. 第一部分内容

### 1.1 具体小节

[分块1处理后的内容...]

## 2. 第二部分内容

[分块2处理后的内容，编号接续前面的结构...]
```

**第四步：最终合并结果**
```markdown
# 文档总标题

### 1.2 其他小节

```

## 注意事项

1. **分块边界处理**：确保不会在句子中间分割，优先选择段落边界
2. **上下文连续性**：后一分块需要了解前一分块的标题结构和编号
3. **完整性验证**：最终结果必须包含原文的所有内容
4. **编号一致性**：标题编号必须严格遵循 1, 1.1, 1.1.1 的格式
5. **语义边界合理性**：确保每个标题都位于自然的语义边界处，主题转换合理
6. **字数要求硬性约束**：
   - 严格遵守每章不少于1500字的要求
   - 严格遵守每节不少于500字的要求
   - 字数不足时优先考虑主题聚合和合并，而非拆分
7. **多主题小节应用**：
   - 鼓励将相似或关联度强的主题合并到同一个小节
   - 支持并列型、递进型、对比型、总分型等组织方式
   - 通过主题聚类满足字数要求，提升内容丰富度
8. **结构人脑友好**：
   - 避免过度细分：防止一章下出现过多小节（建议每章3-6个小节）
   - 确保最小单元：每个标题下至少包含一个完整的论述单元，同时满足字数要求
   - 控制层级深度：建议不超过4层，符合认知习惯
   - **段落内部细分优先**：对于需要细分的短内容，优先在节内通过语义换行实现
9. **实用性优先**：以内容的逻辑完整性和读者理解体验为首要考虑，层级深度服务于内容理解
10. **字数与语义的平衡**：在满足字数要求的前提下，确保标题结构的语义合理性和逻辑连贯性
11. **主题关联度优先**：在章节划分时，优先考虑主题间的语义相似性、逻辑相关性和上下文连贯性
