# Text Warn Logic Flaw Command

基于逻辑谬误识别的智能文本逻辑漏洞检测工具，为每个章节添加逻辑缺陷警告提示。

## Usage

```markdown
/text-warn-logic-flaw "{input_text}"
```

## Parameters

- `input_text`: 需要进行逻辑漏洞检测的完整文本

## Output

检测后的完整文本（markdown格式），每个章节内容后会追加逻辑漏洞警告代码块

## 核心功能

### 1. 逻辑漏洞检测
- **逻辑谬误识别**：基于25种常见逻辑谬误进行全面检测
- **章节级别分析**：逐章分析文本中的逻辑问题
- **智能匹配**：根据文本内容匹配相应的逻辑谬误类型

### 2. 风格模仿警告
- **语言风格保持**：警告内容模仿原文的表达习惯
- **简明扼要提示**：用简洁明了的方式指出问题
- **非确定性表述**：强调"潜在可能"而非绝对判断

### 3. 非侵入式输出
- **内容保护**：不修改任何章节标题和原文内容
- **格式化警告**：使用统一的markdown代码块格式
- **友好提示**：明确标注仅供参考，请读者留意

## 逻辑谬误检测类型

### 因果关系类
1. **因果关系漏洞**：错误假设因果关系
2. **混淆因果和相关性**：将相关性误认为因果关系
3. **忽略其他因素**：遗漏影响结论的重要因素

### 数据论证类
4. **数据和统计漏洞**：使用不准确或不完整数据
5. **过度概括**：基于有限样本得出普遍结论
6. **不合理的比较**：进行比较时存在问题

### 逻辑推理类
7. **假设未验证**：存在未经验证的关键假设
8. **循环论证**：用结论证明结论
9. **滑坡谬误**：假设不可控的连锁反应
10. **不合理的推断**：从事实得出不合理的结论
11. **不合理的逻辑**：整体逻辑结构不合理

### 概念表达类
12. **偷换概念**：在论证中改变关键概念定义
13. **虚假二分**：简化问题为只有两种选择
14. **不合理的定义**：对关键概念定义不当
15. **类比不当**：将不相似情况进行错误类比

### 论证质量类
16. **不完整的论据**：论据不足以支持结论
17. **不相关的论据**：论据与结论无关
18. **不一致的论据**：论据之间存在矛盾
19. **不合理的证据**：使用的证据存在问题
20. **不合理的论证**：整体论证过程有缺陷

### 认知偏误类
21. **诉诸权威**：过度依赖权威意见
22. **诉诸情感**：用情感替代逻辑论证
23. **诉诸无知**：因缺乏证据就认为结论成立

## 检测流程

### 第一步：文本结构分析
1. **章节识别**：分离标题和章节内容
2. **内容解析**：分析每个章节的核心论点和论证过程
3. **风格分析**：识别原文的语言特点和表达习惯

### 第二步：逻辑漏洞扫描
1. **逐类检测**：按照25种逻辑谬误类型逐一扫描
2. **语境分析**：在具体语境中判断是否存在相应问题
3. **严重程度评估**：评估每个逻辑问题对论证的影响程度

### 第三步：警告生成
1. **问题归纳**：将检测到的问题进行分类整理
2. **风格匹配**：用原文的语言风格重述警告内容
3. **位置标记**：确定警告插入的最佳位置

### 第四步：输出整合
1. **原文保持**：完全保留原始文本结构和内容
2. **警告追加**：在每个章节内容后添加逻辑漏洞警告
3. **格式统一**：确保输出格式符合markdown规范

## 警告格式

```text
💡 逻辑漏洞提示（仅供参考）：
本章节可能存在以下逻辑问题：
- [检测到的逻辑问题1，用原文风格简述]
- [检测到的逻辑问题2，用原文风格简述]
- [检测到的逻辑问题3，用原文风格简述]

以上提示旨在帮助读者进行批判性思考，不代表一定存在错误
```

## 检测优先级

### 高优先级
- **因果关系漏洞**：对论证核心有重大影响的因果假设
- **数据统计问题**：支撑关键结论的数据存在缺陷
- **循环论证**：根本性的逻辑结构错误

### 中优先级
- **过度概括**：可能影响结论适用范围的概括问题
- **类比不当**：比较对象在关键特征上不相似
- **忽略其他因素**：遗漏重要的反驳因素

### 低优先级
- **表达细节问题**：不影响核心论证的表达问题
- **假设细节问题**：次要假设的验证问题
