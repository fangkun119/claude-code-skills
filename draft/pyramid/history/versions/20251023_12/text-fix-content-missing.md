# Text Fix Content Missing Command

基于原始文本检查和修复改写后文本内容缺失的智能工具，确保信息完整性。

## Usage

```markdown
/text-fix-content-missing "{rewritten_text}" "{original_text}"
```

## Parameters

- `rewritten_text`: 改写后的文本，可能存在内容缺失
- `original_text`: 原始参考文本，用于检查内容完整性

**错误处理：**
- 参数缺失时返回错误提示
- 支持中英文混合文本处理
- 自动处理文本编码问题

## Output

直接输出修复后的完整文本（纯文本格式，UTF-8编码）

## 核心功能

### 1. 智能内容分析
- **语义对比**：深度分析两段文本的语义内容差异
- **信息完整性检查**：识别改写后文本中遗漏的关键信息
- **结构映射**：建立原始内容与改写内容的对应关系

### 2. 精准内容补充
- **格式保持**：严格按照改写后文本的格式和风格补充内容
- **结构对齐**：将缺失内容插入到对应的位置和结构中
- **标题保护**：绝不修改任何章节标题，只处理章节内容

### 3. 质量保障
- **信息零丢失**：确保原始文本中的所有信息都得到保留
- **逻辑连贯**：保证补充内容后的文本逻辑完整、表达流畅
- **风格统一**：保持改写后文本的写作风格和结构特点

## 执行流程

### 第一步：内容对比分析
1. **语义解析**：分别解析原始文本和改写后文本的核心内容
2. **差异识别**：识别改写后文本中缺失的具体信息点
3. **结构分析**：分析两段文本的章节结构和组织方式

### 第二步：缺失内容定位
1. **内容映射**：确定每个缺失内容在改写文本中的对应位置
2. **结构匹配**：找到缺失内容应该插入的章节和段落
3. **格式适配**：根据改写文本的风格调整补充内容的表达方式

### 第三步：内容补充修复
1. **精准插入**：将缺失内容插入到正确的位置
2. **风格统一**：确保补充内容与改写文本的表达风格一致
3. **逻辑优化**：优化补充内容与原文内容的衔接关系

## 支持的缺失内容类型

### 事实信息类
- **关键数据**：遗漏的数字、统计信息、具体参数
- **时间信息**：缺失的时间点、日期、时间段
- **地点信息**：遗漏的地理位置、场所名称

### 论证结构类
- **论据缺失**：支撑论点的关键例子、证据
- **逻辑链条**：论证过程中的重要推理环节
- **对比对象**：比较分析中的另一面信息

### 描述细节类
- **过程描述**：操作步骤、发展过程的遗漏
- **特征描述**：对象特征、属性的不完整描述
- **背景信息**：理解内容所需的背景资料

### 人物事件类
- **关键人物**：事件中的重要参与者信息
- **事件环节**：事件发展过程中的关键环节
- **因果关系**：导致结果的重要原因和条件

## 补充策略

### 语义适配
- **同义转换**：用改写文本的表达方式重述缺失内容
- **结构调整**：按照改写文本的组织逻辑重新排列信息
- **简化优化**：在保持原意的基础上优化表达方式

### 位置精准
- **段落开头**：核心观点或结论类内容
- **段落中间**：论证过程或说明性内容
- **段落结尾**：总结或过渡性内容

### 风格统一
- **语言风格**：保持与改写文本一致的语气和表达习惯
- **结构特点**：延续改写文本的段落组织和信息呈现方式
- **专业水准**：维持原文的专业性和准确性

### 结构一致性
- [ ] 章节标题完全保持不变
- [ ] 内容补充位置准确合理
- [ ] 文本结构层次清晰
- [ ] 段落组织逻辑合理

### 表达质量
- [ ] 补充内容与改写文本风格一致
- [ ] 语言表达流畅自然
- [ ] 专业术语使用准确
- [ ] 整体阅读体验良好
