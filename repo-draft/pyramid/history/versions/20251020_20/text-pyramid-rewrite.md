# Pyramid Rewrite Command

根据Barbara Minto的《金字塔原理》对**单个章节内容**进行重写，仅改写提供的文本内容，不改变标题结构。

## Usage

```markdown
/doc-pyramid-rewrite "章节内容文本" "额外需求"
```

## Parameters

- `chapter_content_text`: 需要重写的章节内容（纯文本，非文件路径）
- `additional_requirements`: 额外的重写需求（可选，如果没有则为空字符串）

## Output

该命令将重写后的内容直接输出到控制台，不会修改输入文件或创建新文件。

## 核心功能

### 1. 智能内容识别
- **论述性内容**：逻辑论证结构优化
- **叙事性内容**：时间顺序和故事结构整理
- **说明性内容**：层次结构和步骤说明规范化
- **混合内容**：分类处理不同段落性质
- **需求适配**：根据用户额外需求调整重写风格和重点

### 2. 金字塔原理应用

#### 结论先行（必须执行）
```
每个段落开头：**核心结论加粗显示**
准确概括全文信息，避免模糊表述
```

#### 强制列表转换（核心功能）
**检测标准：**
- 并列连词：和、与、及、同时
- 序列词汇：首先、其次、第三、最后
- 举例用语：例如、比如、具体来说
- 分类描述：特点、优势、问题、方案

**转换要求：**
- **3个及以上并列内容→必须转换为列表**
- **目标：70%+内容以列表形式呈现**
- 使用适当引导语（"主要包括："、"优势如下："）

#### 逻辑层次构建
```
一级：核心结论（加粗）
二级：主要论据（列表格式）
三级：具体例证（子列表支持）
```

### 3. 局部论述结构保留与凸显

#### 论述结构识别与保护
- ✅ **论述方法识别**：自动识别18种局部论述结构
- ✅ **凸显词汇保留**：保留原文中凸显论述结构的关键词汇
- ✅ **逻辑关系保护**：保持原有的论证逻辑关系
- ✅ **论证强度维持**：确保不降低原有的说服力

#### 凸显论述结构的词汇库

**基础逻辑类凸显词汇：**
- **逻辑推理**：基于...可以推断出、进一步推理、由此可证、因此、由此可见
- **因果分析**：究其原因、深层次分析表明、这导致了...、其根源在于...、根本原因是...
- **反证法**：假设相反、退一步讲、如果...那么...显然不合理、因此，原观点成立
- **逻辑结构清晰**：首先...其次...最后...、在此基础上、更进一步说、综上所述

**证据支撑类凸显词汇：**
- **具体例子**：以...为例、具体来说、这个例子清楚地表明...、这个例子充分证明了...
- **数据统计**：数据显示...、统计表明...、具体来看...、进一步分析数据发现...、这些数字充分说明...
- **研究引用**：根据...研究显示、...报告指出、研究进一步表明...、研究结果证实了...
- **案例实例**：回顾...案例、以...为例证、案例表明...、从中我们可以总结出...、这个案例充分说明了...

**对比分析类凸显词汇：**
- **对比分析**：相比之下、与此不同的是、通过对比可以发现、前者...而后者...、...的优势在于...、因此，...更具有优势
- **类比论证**：正如...一样、这类似于...、同样地...、相似地...、这个类比告诉我们...

**权威支撑类凸显词汇：**
- **权威引用**：正如...所言、...权威机构指出、专家认为...、权威人士强调...、因此，...的观点具有权威性
- **专业观点**：根据...理论、在...领域看来、理论上讲...、专业角度分析...、理论支撑了这一观点

**前瞻性分析类凸显词汇：**
- **预测未来**：展望未来、可以预见、随着...的发展、趋势表明...、这预示着...
- **背景趋势**：从宏观背景看、当前趋势显示、在此背景下...、趋势进一步表明...、大环境决定了...

**价值理念类凸显词汇：**
- **道德伦理**：从道德角度看、伦理上讲、这符合...、这违背了...、因此，...是正确的选择

**修辞表达类凸显词汇：**
- **修辞手法**：如同...一样、可以说是...、不仅是...更是...甚至是...、由此可见...
- **情感诉求**：我们有理由相信、这让我们深感...、令人欣慰的是...、这激励着我们...、因此，我们应该...

**论证完善类凸显词汇：**
- **反驳反对**：有人可能会说、也许有人质疑、但是，这种观点忽视了...、因此，这种质疑是不成立的

### 4. 内容保护原则
- ✅ **零信息丢失**：保持所有原始信息完整
- ✅ **零内容添加**：不增加原文不存在的内容
- ✅ **数据准确性**：保持事实、数据、引用不变
- ✅ **源代码保护**：不改写任何代码块
- ✅ **论述结构保留**：保留原文的论述结构标识和凸显词汇
- ✅ **需求满足**：在满足金字塔原理基础上尽量满足用户额外需求

## 列表格式规范

### 有序列表（1. 2. 3.）
**使用场景：**
- 时间顺序的步骤描述
- 重要性排序的观点列举
- 因果关系的发展过程

### 无序列表（-）
**使用场景：**
- 并列特点描述
- 同等级别的观点列举
- 分类对比分析

### 多级列表示例
```markdown
**核心结论**

**主要分析：**
- **观点1**：详细解释
  - 具体例子1
  - 具体例子2
- **观点2**：详细解释
  - 具体例子1
  - 具体例子2
```
