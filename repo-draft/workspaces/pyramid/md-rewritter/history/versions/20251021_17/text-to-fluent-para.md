# Text To Fluent Paragraph Command

基于金字塔原理的文本流畅化工具，将列表式文本转换为连续流畅的文字表达。

## Usage

```markdown
/text-to-fluent-para "{input_text}"
```

## Parameters

- `input_text`: 包含有序/无序列表的完整文本

## Output

直接输出流畅化后的连续文本（纯文本格式，UTF-8编码）

## 核心功能

### 1. 结构识别
- **列表检测**：自动识别有序列表、无序列表和层级结构
- **核心结论提取**：识别每个段落的核心观点和结论
- **逻辑层次分析**：分析核心结论→主要论据→具体例证的层次关系

### 2. 流畅化转换
- **列表消解**：将列表内容转换为连贯的文字表达
- **连接优化**：添加自然的过渡词和连接短语
- **表达优化**：提升文本的连贯性和可读性

### 3. 金字塔结构保持
- **结论先行**：保持每个段落的结论领先结构
- **逻辑层次**：保持核心观点→支撑论据→具体例证的逻辑顺序
- **信息完整**：确保所有重要信息不丢失

## 金字塔原理应用

### 结构层次
1. **核心结论**：段落或章节的主要观点
2. **主要论据**：支撑核心结论的关键理由
3. **具体例证**：支撑主要论据的具体事实、数据或例子

### 转换原则
- **列表转段落**：将列表项转换为连贯的句子
- **层次扁平化**：将多层级结构转为线性叙述
- **连接词优化**：使用自然的过渡表达保持逻辑连贯

## 列表转换策略

### 有序列表转换
**原格式：**
1. 第一个观点
2. 第二个观点
3. 第三个观点

**转换后：**
首先，第一个观点；其次，第二个观点；最后，第三个观点。

### 无序列表转换
**原格式：**
- 观点A
- 观点B
- 观点C

**转换后：**
观点A、观点B以及观点C都是重要考量。

### 多层列表转换
**原格式：**
1. 主要观点
   - 次要观点A
   - 次要观点B
2. 另一个主要观点
   - 次要观点C

**转换后：**
首先，主要观点，具体来说包括次要观点A和次要观点B。其次，另一个主要观点，特别体现在次要观点C方面。

## 处理流程

### 第一步：结构分析
1. **章节识别**：分离标题和章节内容
2. **列表检测**：识别各种类型的列表结构
3. **层级分析**：确定信息的层次关系

### 第二步：转换规划
1. **逻辑梳理**：重新组织信息的逻辑顺序
2. **连接词选择**：选择合适的过渡表达方式
3. **流畅性优化**：考虑文本的表达节奏

### 第三步：文本重构
1. **列表消解**：将列表转换为连贯的文字
2. **连接词插入**：在适当位置添加过渡表达
3. **流畅性优化**：确保整体表达自然流畅

### 第四步：质量检查
1. **信息完整性**：确保所有信息都得到保留
2. **逻辑清晰性**：验证表达逻辑清晰易懂
3. **阅读流畅性**：检查是否符合连续文本的表达需求
