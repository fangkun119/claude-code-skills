# Text To Fluent Paragraph Command

基于金字塔原理的文本流畅化工具，将列表式文本转换为连续流畅的文字表达。

## Usage

```markdown
/text-to-fluent-para "{input_text}"
```

## Parameters

- `input_text`: 包含有序/无序列表的完整文本

## Output

直接输出流畅化后的连续文本（纯文本格式，UTF-8编码）

## 核心功能

### 1. 结构识别
- **列表检测**：自动识别有序列表、无序列表和层级结构
- **核心结论提取**：识别每个段落的核心观点和结论
- **逻辑层次分析**：分析核心结论→主要论据→具体例证的层次关系

### 2. 流畅化转换
- **列表消解**：将列表内容转换为连贯的文字表达
- **精准连接**：在真正需要时添加过渡词，避免过度机械化
- **表达优化**：提升文本的连贯性和可读性，增强自然度

### 3. 灵活结构处理
- **结构适配**：根据内容类型灵活处理金字塔结构
- **文体适应**：针对学术、商业、叙述等不同文体调整策略
- **信息完整**：确保所有重要信息不丢失

## 金字塔原理应用

### 结构层次
1. **核心结论**：段落或章节的主要观点
2. **主要论据**：支撑核心结论的关键理由
3. **具体例证**：支撑主要论据的具体事实、数据或例子

### 转换原则
- **列表转段落**：将列表项转换为连贯的句子
- **层次扁平化**：将多层级结构转为线性叙述
- **精准连接**：在真正需要时使用过渡表达，避免过度连接
- **文体适配**：根据文本类型采用不同的流畅化策略

## 列表转换策略

### 有序列表转换
**原格式：**
1. 第一个观点
2. 第二个观点
3. 第三个观点

**灵活转换策略：**
- **简洁版**：第一个观点、第二个观点和第三个观点。
- **叙述版**：第一个观点，同时第二个观点，最后第三个观点。
- **逻辑版**：首先，第一个观点；其次，第二个观点；最后，第三个观点。

**选择依据**：根据文体和上下文需要选择最自然的表达方式。

### 无序列表转换
**原格式：**
- 观点A
- 观点B
- 观点C

**灵活转换策略：**
- **简洁版**：观点A、观点B和观点C。
- **强调版**：观点A、观点B以及观点C都是重要方面。
- **叙述版**：包括观点A，同时还有观点B和观点C等考虑因素。

**选择依据**：根据重要性和表达需要选择合适的连接方式。

### 多层列表转换
**原格式：**
1. 主要观点
   - 次要观点A
   - 次要观点B
2. 另一个主要观点
   - 次要观点C

**灵活转换策略：**
- **简洁版**：主要观点包括次要观点A和次要观点B，另一个主要观点则体现在次要观点C方面。
- **叙述版**：首先，主要观点，这包括两个重要方面：次要观点A和次要观点B。然后，另一个主要观点，特别是次要观点C方面。
- **逻辑版**：一方面，主要观点，具体包括次要观点A和次要观点B；另一方面，另一个主要观点，主要体现在次要观点C上。

**选择依据**：根据信息层次和表达重点选择合适的方式。

## 处理流程

### 第一步：结构分析
1. **章节识别**：分离标题和章节内容
2. **列表检测**：识别各种类型的列表结构
3. **层级分析**：确定信息的层次关系

### 第二步：转换规划
1. **文体识别**：判断文本类型（学术/商业/叙述/技术等）
2. **逻辑梳理**：重新组织信息的逻辑顺序
3. **连接词评估**：评估是否真的需要过渡词，避免过度使用
4. **流畅性规划**：考虑文本的表达节奏和自然度

### 第三步：文本重构
1. **列表消解**：将列表转换为连贯的文字
2. **精准连接**：仅在真正需要时添加过渡表达
3. **句式优化**：创造长短句搭配，避免单调重复
4. **流畅性优化**：确保整体表达自然流畅

### 第四步：质量检查
1. **信息完整性**：确保所有信息都得到保留
2. **逻辑清晰性**：验证表达逻辑清晰易懂
3. **自然度检查**：检查文本是否流畅自然，无机械感
4. **文体一致性**：确保转换后符合目标文体的特征
5. **阅读流畅性**：检查是否符合连续文本的表达需求

## 连接词参考

### 顺序连接
- **首先...其次...再次...最后**
- **第一...第二...第三**
- **一方面...另一方面**
- **不仅...而且...更重要的是**

### 转折连接
- **虽然...但是**
- **尽管...然而**
- **反之**
- **与此不同**

### 因果连接
- **因此**
- **由于**
- **正因为如此**
- **由此导致**

### 补充连接
- **此外**
- **同时**
- **另外**
- **同样重要的是**
