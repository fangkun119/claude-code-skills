# Document Fluent Command

基于金字塔原理的文档流畅化工具，将 markdown 文件中的章节内容重写为流畅的语言表达。

## Usage

```markdown
/doc-fluent "{input_file_path}"
```

## Parameters

- `input_file_path`: 输入的 markdown 文件路径

## Output

改写后完整的 markdown 内容打印到标准输出（纯文本格式，UTF-8编码）

## 核心功能

### 1. 文档结构处理
- **章节识别**：自动识别 markdown 文档的章节标题和内容
- **标题保护**：保持所有章节标题完全不变
- **内容分离**：将章节内容从标题中分离出来进行处理

### 2. 内容流畅化重写
- **列表消解**：将列表结构转换为连贯的文字表达（仅在要求流畅化时）
- **表达优化**：提升文本的连贯性和可读性
- **信息完整**：确保所有信息不添加、不删减，与说明/论述/叙述目标的信息零丢失
- **调整范围**：调整范围仅限于表达方式和表达风格，不改变实质内容
- **智能判断**：根据要求语义决定是否保持列表化风格
- **示例保护**：不修改任何代码示例、操作演示、命令行展示等专业示例

### 3. 文档重构输出
- **结构保持**：保持原有的 markdown 结构层次
- **标题不变**：所有章节标题保持原样
- **内容替换**：仅替换章节内容的表达形式

## 执行流程

### 第一步：文档读取
1. **文件读取**：读取指定的 markdown 文件内容
2. **结构解析**：识别文档中的章节标题和内容
3. **内容分离**：将标题和内容分别处理

### 第二步：内容处理
1. **章节遍历**：逐个处理每个章节的内容
2. **需求分析**：分析用户要求，判断是否需要将列表转换为流畅表达
3. **列表检测**：识别内容中的各种列表结构
4. **智能重写**：
   - 如果要求包含"流畅"、"适合朗读"等语义：将列表转换为流畅文字表达
   - 如果不包含：保持列表化风格，仅调整每行措辞和句式
5. **信息保护**：确保与说明/论述/叙述目标的信息零丢失

### 第三步：文档重构
1. **标题保留**：保持所有章节标题不变
2. **内容替换**：用流畅化的内容替换原有内容
3. **结构重组**：重新组合成完整的 markdown 文档

### 第四步：输出结果
1. **标准输出**：将完整的 markdown 内容输出到标准输出
2. **文件保护**：不修改原始输入文件
3. **格式保持**：保持 markdown 格式的完整性

## 流畅化转换策略

### 智能处理原则
根据用户要求的语义智能判断处理方式：

**需求语义分析：**
- **流畅化需求**：当要求包含"流畅"、"适合朗读"、"自然表达"等语义时，执行列表转换
- **保持结构需求**：当要求不包含上述语义时，保持列表化风格，仅优化措辞

### 有序列表转换（仅在流畅化需求时）
根据内容类型选择合适的转换方式：
- **简洁版**：直接用"、"和"等简单连接
- **逻辑版**：使用"首先、其次、最后"等逻辑连接
- **叙述版**：融入叙述中，自然过渡

### 无序列表转换（仅在流畅化需求时）
避免过度连接，选择最自然的方式：
- **简单并列**：用"、"、"和"连接同等重要的项目
- **重点强调**：需要强调时，使用"特别是"、"尤其是"等
- **省略连接**：当内容本身逻辑清晰时，可以直接并列

### 多层列表转换（仅在流畅化需求时）
保持层次关系的清晰性：
- **层次展开**：用"包括"、"体现在"等保持层次关系
- **细节补充**：用"具体来说"、"例如"等展开细节
- **自然融合**：避免生硬的结构词，保持叙述流畅

### 列表风格优化（保持结构需求时）
当不执行列表转换时，专注于：
- **措辞优化**：改进每项的表达方式和用词
- **句式调整**：优化句子结构，使其更符合语言习惯
- **信息完整性**：确保与说明/论述/叙述目标的信息零丢失
- **风格统一**：保持整体表达风格的一致性
