# Doc Warn Logic Flaw Command

基于逻辑谬误识别的智能文档逻辑漏洞检测工具，为整个 markdown 文档添加逻辑缺陷分析章节。

## Usage

```markdown
/doc-warn-logic-flaw "{input_file_path}.md"
```

## Parameters

- `input_file_path.md`: 需要进行逻辑漏洞检测的 markdown 文件路径

## Output

包含逻辑漏洞分析结果的完整 markdown 文档，直接输出到标准输出

## 核心功能

### 1. 文档级逻辑检测
- **全文分析**：对整个 markdown 文档进行综合逻辑分析
- **跨章节关联**：检测文档各章节间的逻辑一致性
- **整体论证评估**：评估整篇文档的论证结构和逻辑链条

### 2. 逻辑谬误识别
- **25种谬误检测**：基于完整的逻辑谬误类型库进行全面检测
- **语境理解**：在文档整体语境中判断逻辑问题的严重程度
- **关联分析**：识别不同章节间可能存在的逻辑矛盾

### 3. 智能章节生成
- **自动定位**：在文档末尾添加逻辑分析章节
- **结构化输出**：按逻辑谬误类型组织检测结果
- **标准输出**：将完整结果直接输出到标准输出
- **友好提示**：强调分析结果仅供参考

## 逻辑谬误检测类型

### 因果关系类
1. **因果关系漏洞**：错误假设因果关系
2. **混淆因果和相关性**：将相关性误认为因果关系
3. **忽略其他因素**：遗漏影响结论的重要因素

### 数据论证类
4. **数据和统计漏洞**：使用不准确或不完整数据
5. **过度概括**：基于有限样本得出普遍结论
6. **不合理的比较**：进行比较时存在问题

### 逻辑推理类
7. **假设未验证**：存在未经验证的关键假设
8. **循环论证**：用结论证明结论
9. **滑坡谬误**：假设不可控的连锁反应
10. **不合理的推断**：从事实得出不合理的结论
11. **不合理的逻辑**：整体逻辑结构不合理

### 概念表达类
12. **偷换概念**：在论证中改变关键概念定义
13. **虚假二分**：简化问题为只有两种选择
14. **不合理的定义**：对关键概念定义不当
15. **类比不当**：将不相似情况进行错误类比

### 论证质量类
16. **不完整的论据**：论据不足以支持结论
17. **不相关的论据**：论据与结论无关
18. **不一致的论据**：论据之间存在矛盾
19. **不合理的证据**：使用的证据存在问题
20. **不合理的论证**：整体论证过程有缺陷

### 认知偏误类
21. **诉诸权威**：过度依赖权威意见
22. **诉诸情感**：用情感替代逻辑论证
23. **诉诸无知**：因缺乏证据就认为结论成立

## 检测流程

### 第一步：文档解析
1. **文件读取**：完整读取指定的 markdown 文件
2. **结构分析**：识别文档的章节结构和内容组织
3. **主题识别**：理解文档的核心主题和论证目标

### 第二步：全文扫描
1. **整体论证分析**：分析整篇文档的论证结构和逻辑链条
2. **跨章节一致性检查**：检查各章节间的逻辑一致性
3. **关键论点识别**：识别文档中的核心论点和支撑论据

### 第三步：逻辑谬误检测
1. **逐类扫描**：按照25种逻辑谬误类型对全文进行检测
2. **严重程度评估**：评估每个检测到的问题对文档整体论证的影响
3. **问题分类整理**：将检测到的问题按类型和严重程度分类

### 第四步：结果整合
1. **章节生成**：在文档末尾添加"逻辑漏洞分析"章节
2. **结果组织**：按逻辑谬误类型组织检测结果
3. **标准输出**：将处理后的完整 markdown 内容直接输出到标准输出
4. **友好提示**：添加免责声明，强调分析结果仅供参考

## 检测优先级

### 高优先级（重点报告）
- **整体逻辑结构问题**：影响整篇文档论证结构的根本性问题
- **关键假设未验证**：支撑核心结论的关键假设缺乏验证
- **循环论证**：在核心论证中存在的循环逻辑问题

### 中优先级（详细分析）
- **数据统计缺陷**：支撑重要结论的数据存在质量问题
- **过度概括**：影响结论适用范围的概括问题
- **跨章节逻辑矛盾**：不同章节间的逻辑不一致问题

### 低优先级（简要提及）
- **表达细节问题**：不影响核心论证的表达问题
- **次要逻辑问题**：对整体论证影响较小的逻辑瑕疵

## 重要提醒

- **文件路径**：确保提供的 markdown 文件路径正确且文件存在
- **文档保护**：绝不修改原始文档的任何内容
- **章节添加**：只在文档末尾添加新的分析章节
- **客观分析**：保持客观中立的表述方式
- **友好提示**：明确标注分析结果仅供参考
- **批判性思考**：旨在帮助读者进行批判性思考，不代表一定存在错误
