## 预期结果
- 不完整代码块：
  - 脚本应优雅地处理没有关闭标记的代码块
  - 不应因缺少结束标记而崩溃
- 多重嵌套：
  - 应正确区分不同数量的反引号
  - 只有匹配数量的反引号才能关闭代码块
- 特殊字符：
  - 转义字符应被正确处理
  - HTML 实体应被正确解析
  - 特殊空白字符不应破坏标题识别
- 格式边界：
  - 空文件应返回空结果
  - 纯代码块文件应返回空标题列表
  - 标题和代码块边界应被准确定位
- 缩进处理：
  - 3个空格以内的缩进应被识别为标题
  - 4个空格的缩进应被视为代码块
- 异常格式：
  - 不匹配的代码块标记应被正确处理
  - 标题中的反引号不应触发代码块识别
- 编码字符：
  - 所有 Unicode 字符应被正确支持
  - 不应出现编码相关的错误
- 重复编号：
  - 重复的编号应被检测为格式错误
  - 脚本应提供清晰的错误信息
