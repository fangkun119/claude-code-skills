

# 用户初始需求如下：

## 目标

制作一篇精炼实用的AI编程教程。

## 受众需求

目标受众：学习AI编程的工程师
受众需求：想掌握AI编程方法论、具备指导AI做项目的架构思维。具体来说，例如了解AI编程的各个环节中、步骤是什么、注意事项是什么、要如何思考、考虑哪些问题、要做哪些事情……等等

## 内容组成 - 分两部分

不用保持原文的顺序，按照下面两部分组织内容

### 第一部分（前几个章节）- 方法论提炼

**目标**：focus在提炼AI编程方法论、即思考分析和做项目的方法、关注点、操作要领等，提供一份简明扼要、但清晰易懂的参考手册

- 这部分用于：方法指导、快速复习、项目Check List
- 这部分内容更像一个参考的手册，帮助受众在项目对应的阶段快速参考，知道该阶段该如何去做。例如、在某个阶段、需要达成什么目标、做什么事情、做这件事情时要如何思考、思考过程中要关注哪些问题、有哪些方法、注意点和技巧是什么 …… 等等
- 这一部分不深入到具体的技术栈。

### 第二部分（后几个章节）- 实战演示

**目标**：结合实践，帮助读者深入理解第一部分的内容。

这一部分：

- 让第一部分的方法论能够结合实践，结合原文中的例子、实操、所用到的技术栈、项目背景等等，复现原文的项目过程。
- 它是更加深入的教材，让受众”不仅知其然、也知其所以然“
- 它是更加结合实操案例演示的教材，让受众能够在方法论的基础上，通过观看具体的实操案例，获得更好的理解

## 注意事项

你需要先站在受众需求的角度，围绕AI编程方法论，围绕各部分的目标，主导思考什么内容是高价值的，进行内容选择和组织。
